§ 1 — The Problem

Can a subset of these numbers hit exactly t?

Given a set S = {x₁, …, xₙ} and a target t, decide whether some subset of S sums to t. This is NP-complete. Not because the recurrence is hard — but because t can be exponentially large in the input size.

A tiny example

Let S = {1, 3, 9} and t = 4. Is there a subset summing to 4? Yes: {1, 3}. ✓
What about t = 5? No subset works. ✗
What about t = 13? Yes: {1, 3, 9}. ✓

The decision-vs-optimization twist. The optimization version asks: find the largest subset sum ≤ t. That's what the FPTAS approximates — the algorithm returns a value z with y/z* ≤ (1+ε), where z* is the true optimum.
The exact recurrence
Sub(A, k, last): if (k < 0 or last < 0) return FALSE if (k == 0) return TRUE return Sub(A, k, last-1) // skip A[last] OR Sub(A, k - A[last], last-1) // take A[last]

At every element you either take it (target shrinks by its value) or skip it (target unchanged). Two children per call → 2ⁿ leaves.

Same recurrence, no recomputation: a grid of booleans.

Why this is O(n·t), not polynomial. The grid has n·t cells. Looks linear — but t is given as a number, whose input size is only log t bits. So O(n·t) = O(n·2log t) — exponential in the input size. This is why your notes wrote "NP-complete" next to the grid: pseudo-polynomial, not polynomial.
§ 3 — The Trim Trick

Throw away near-duplicates. Keep the list logarithmic.

The exact algorithm builds a list of every reachable sum. After i rounds the list can have 2i values. Trim(L, δ) walks L in sorted order and drops any y that's within a factor (1+δ) of the previously kept anchor — an already-kept value z stands in as a close-enough substitute.

§ 4 — Page 2: Why the Error Stays Small

Choose δ = ε / 2n. Compounding gives back 1 + ε.

We trim n times (once per element). Each trim multiplies the worst-case error by (1+δ). After n rounds the error has compounded — but the tight choice of δ keeps the total within (1+ε).

Step 1 — pick δ
δ = ε / 2n
Why divided by 2n? Because (1+x)n ≈ enx — and we want the eventual exponent to be ε/2, not ε. The "2" buys headroom for the next step.
Step 2 — compound n times
(1 + ε/2n)n ≤ eε/2
Standard inequality (1+x) ≤ ex applied n times. n rounds of trimming = at most this multiplicative slack.
Step 3 — bound the exponential
eε/2 ≤ 1 + ε/2 + (ε/2)²
The other half of the inequality: ex ≤ 1 + x + x². Lets us turn the exponential back into a clean (1 + something).
Step 4 — drop the tail
≤ 1 + ε
Since ε < 1, the (ε/2)² term is at most ε/2. Sum: ε/2 + ε/2 = ε. Total error bound = (1 + ε). ∎
Why is (ε/2)² ≤ ε/2 when ε < 1? Squaring a number less than 1 makes it smaller, not bigger. Since ε < 1, we have ε/2 < ½, and squaring a fraction shrinks it further. The algebra: (ε/2)² = ε²/4 ≤ ε/2 ⟺ ε ≤ 2, which holds the moment ε < 1. Concrete:

§ 5 — Page 3: Runtime, and Why "FPTAS"

List size stays O(n log t / ε). Total work polynomial in n, log t, 1/ε.

The bound on list length is what makes this fast. Trimmed values increase by at least a factor of (1+δ), so a list whose values lie in [0, t] holds at most log1+δ t of them.

rounds
n
One trim per element of S.
list size per round
O(n · log t / ε)
From log1+δ t with δ = ε/2n. Use ln(1+x) ≥ x/(1+x) to flatten.
total work
O(n²·log t / ε)
Multiply rounds × per-round merge+trim. Polynomial in n, log t, and 1/ε.
What "fully" means in FPTAS. A PTAS is polynomial in n for any fixed ε. A fully PTAS is polynomial in n and 1/ε together. Your circled remark is exactly the boundary:
· ε > 1/n → 1/ε is small → algorithm is polynomial ✓
· ε ≈ 1/t (i.e. exact) → 1/ε is exponential in input size → back to NP-hard ✗
The FPTAS gives you a knob between accuracy and tractability.

The unstated transition between pages. Between page 1 and page 2 there's an unwritten line the professor probably said aloud: "we can't solve this exactly in poly-time, so let's accept ε% error and see how fast we can go." ε is not derived from anywhere — it's the user's input, the dial you turn. Pages 2 and 3 then prove the algorithm honors that dial: error ≤ (1+ε), runtime polynomial in 1/ε.
